# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Changed

#### All packages
- Renamed governance artifact `.cursorrules` → `.iderules` across CLI, web handbook, and VS Code extension. The file is IDE-agnostic; references to Cursor as an IDE remain unchanged.

### Added

#### Web Handbook
- Template Marketplace (`/templates`) — official + community templates with downloads, copy, upvotes and GitHub submissions
- Community Prompt Library (`/community-prompts`) — browse curated + community prompts, upvote with localStorage persistence, filter by category/tool/stack, and submit new prompts via GitHub issue template

#### GitHub Actions
- `vibe-check` composite action in `.github/actions/vibe-check` + dogfood workflow `.github/workflows/vibe-check.yml` — runs `vibe check --strict` and `vibe review --base origin/main` in CI

#### CLI (`@vibecoding/cli`)
- `vibe init` and `vibe context` now auto-detect `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` and generate a project-specific `AGENTS.md` from the detected stack summary using the LLM; falls back to local heuristics when no key is present or the API fails
- `vibe init` now merges into an existing `AGENTS.md` by default (preserves user content outside the managed markers); add `--overwrite` to replace the file completely
- `vibe context` now merges into an existing `AGENTS.md` by default; add `--overwrite` to replace the file completely
- `vibe init --no-llm` and `vibe context --no-llm` — force local heuristic generation and skip all LLM calls
- `vibe init --overwrite` and `vibe context --overwrite` — replace an existing `AGENTS.md` instead of merging
- `vibe init --name <name>` — override the detected project name
- Smarter policy generation for Security, Testing, and Deployment policies based on the detected auth provider, test framework, and framework
- `vibe init --describe` and `vibe context --describe` — generate `AGENTS.md` from a natural-language project description using OpenAI/Anthropic (native fetch, no SDK dependencies; `VIBE_PROVIDER` / `VIBE_MODEL` overrides; fallback to heuristic generation if the API fails)
- `vibe chat` AI mode — when `OPENAI_API_KEY` or `ANTHROPIC_API_KEY` is set, the plan and optimized prompt are generated by the LLM (native fetch, no SDK dependencies; `VIBE_PROVIDER` / `VIBE_MODEL` overrides; automatic fallback to local heuristics)
- `vibe review --fix` — removes standalone `console.log(...)` statements automatically and prints fix suggestions for the remaining issues
- `vibe review --base <ref>` — review files changed since a git ref (e.g. `origin/main`)
- `vibe review --strict` — exit code 1 when errors or warnings are found (CI gating)

#### VS Code Extension (`vibecoding`)
- `Vibe: Initialize Project` and `Vibe: Refresh Context` now delegate to `@vibecoding/cli` when available, matching the CLI behavior: LLM-generated context when API keys are present, merge-by-default into existing `AGENTS.md`, and fallback to built-in heuristics when the CLI is not installed
- Migrated generated governance directory from `.the-vibecoding-handbook/` to `.vibecoding/` to align with the CLI
- Added `the-vibecoding-handbook.useLLM` setting to enable/disable LLM calls from the extension
- Added `--name` flag to CLI `init` so the extension can pass the project name explicitly
- Multi-language support (i18n) — English, Spanish, Portuguese, Chinese with `vibecoding.changeLanguage` command
- Auto-detects VS Code locale and applies translations to all UI elements (commands, tree views, messages, panel content)

## [1.0.0] - 2026-07-15

### Added

#### Web Handbook
- Interactive landing page with WebGL Neural Aurora shader effect
- 10-chapter documentation (Basics through Advanced Patterns)
- Prompt Library with 50+ curated prompts and intent detection
- Tool Comparison page (Cursor, Copilot, Claude, v0, Replit, etc.)
- Template Generator with ZIP download
- Project Decision Wizard (9-step interactive framework)
- Prompt Optimizer with side-by-side comparison
- Custom cursor with mix-blend-mode
- Konami code easter egg
- Lazy loading with code splitting (321KB bundle)

#### CLI (`@vibecoding/cli`)
- `vibe init` — Initialize project governance files
- `vibe context --auto` — Smart codebase scanner (15+ frameworks)
- `vibe review` — Code review against project policies
- `vibe optimize` — Prompt optimization with intent detection
- `vibe chat` — Interactive planning session
- `vibe check` — Project validation
- `vibe sync` — Template and prompt library sync

#### VS Code Extension
- Policy sidebar tree view
- Decision (ADR) sidebar tree view
- Stack detection sidebar
- Context preview webview panel
- Initialize, Check, and Optimize commands
